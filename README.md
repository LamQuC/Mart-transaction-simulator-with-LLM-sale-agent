# AEPE Transaction Simulator & AI-Driven Store Analytics

## Overview

This project simulates a real-world retail business workflow, including transaction generation, data warehousing, AI-driven price management, and real-time analytics dashboard. It is designed for Data Engineering best practices, using a modern data stack: Python, PostgreSQL, dbt, and Streamlit.

---

## Data Pipeline Flow

1. **Simulation (Bronze Layer)**
	- `src/simulation/customer.py` and `src/simulation/supplier.py` generate all transactional data (orders, inventory changes, logs) into PostgreSQL.
2. **AI Manager (Bronze Layer)**
	- `src/core/llm_manager.py` reads market data, uses an LLM (Ollama or similar) to suggest price changes, and logs decisions to `ai_decision_history`.
3. **Data Modeling (dbt Silver/Gold)**
	- dbt models in `transform/models/` clean, join, and aggregate data for analytics and ML features.
4. **Dashboard (Gold Layer)**
	- `src/dashboard/app.py` provides a real-time Streamlit dashboard for business monitoring and AI decision tracking.

## Setup & Installation

### 1. Prerequisites

- Python 3.9+
- PostgreSQL (local or Docker)
- dbt (v1.11+)
- Node.js (for dbt if needed)
- [Ollama](https://ollama.com/) or compatible LLM API (for AI Manager)
- Docker (optional, for easy service orchestration)

### 2. Clone & Install

```bash
git clone <your-repo-url>
cd transaction-simulator
python -m venv .venv
. .venv/Scripts/activate  # (Windows) or source .venv/bin/activate (Linux/Mac)
pip install -r requirement.txt
```

### 3. Configure Environment

Edit `.env` in the project root:

```
DB_HOST=localhost
DB_PORT=5433
DB_NAME=aepe_db
DB_USER=lam_admin
DB_PASS=lam_admin_pass

OLLAMA_HOST=http://localhost:11434
MODEL_NAME=qwen2.5:1.5b
```

### 4. Start Database

- **With Docker Compose:**
  ```bash
  docker-compose -f deploy/docker-compose.yml up -d
  ```
- **Or manually start PostgreSQL** and create the database.

### 5. Initialize Database Schema

```bash
psql -h $DB_HOST -U $DB_USER -d $DB_NAME -f transform/migrations.sql
```
Or run the seeding script:
```bash
python data/seed_data.py
```

### 6. Run Simulations

Open two terminals and run:
```bash
python src/simulation/customer.py
python src/simulation/supplier.py
```

### 7. Run AI Manager

```bash
python src/core/llm_manager.py
```
> Ensure your Ollama/LLM API is running and accessible.

### 8. Build Analytics Models

```bash
cd transform
dbt run
```

### 9. Launch Dashboard

```bash
streamlit run src/dashboard/app.py
```

## Troubleshooting

- **DB Connection Errors:** Check `.env` and ensure PostgreSQL is running on the correct port.
- **AI 404 Errors:** Make sure Ollama or your LLM API is running and the endpoint matches `OLLAMA_HOST`.
- **dbt Errors:** Ensure all models reference the correct columns (see `int_transaction_details` for schema).
- **Streamlit Errors:** Use `st.rerun()` instead of deprecated `st.experimental_rerun()` if needed.
